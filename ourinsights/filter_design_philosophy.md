# 筛选设计哲学
*2026-05-19*

---

## 核心认识论立场

这个筛选任务的本质不是"找出最好的论文"，而是"对当前 ML 理论空间做覆盖采样"。

两者的区别在操作上是真实的：排名-取前 N 会给你 1500 篇风格相同的统计学习理论论文——因为它们在任何合理的评分轴上都会胜出。覆盖采样要的是不同类型的理论价值，包括被主信号系统性低估的：理论突破但写作不完整的论文、使用深度数学工具但证明有漏洞的论文、认识论意图极强但尚未完成的论文。

这个区别决定了所有后续的设计选择。

---

## 第一阶段：集合论筛选的本质和边界

### 它能做什么

JSON 规则集（rules_jes / rules_claude / rules_codex / rules_danchaofan）是在评分空间里切超矩形。这类筛选有一个重要优点：**可审查性**。每一条规则对应一个可以被质疑、被验证的论断，比如"er > 4 且 tea < 6.5 的论文不值得读"。

对于明确的两端（纯数学论文 vs 纯实验论文），这个方法有效且高效。数据的双峰结构是真实的——ML 论文世界在"理论型 vs 实验型"上存在结构性分裂，中间地带论文极少。

### 它不能做什么

超矩形在欧氏空间里切割。它没有办法表达：
- "这篇论文的 mr 分数相对于它的 tn 分数来说太高了" （模式异常）
- "这篇论文在所有 partial 论文里排在前 30%"（相对位置）
- "mr、tn、md 在这个数据集上是同一件事的三个测量，不是三件独立的事"（维度相关结构）

这三个限制分别对应三个设计方向。

---

## 第二阶段：三个超出超矩形的机制

### 机制一：维度一致性检测

核心思想：**得分模式的内部矛盾本身是信息**。

一篇 mr=9 但 tn=1 和 md=1 的论文，其内部信号是自相矛盾的——极高的数学严谨度不可能与接近零的理论新颖度和数学深度共存。这种组合对应一个真实的噪音模式：把方程式当作装饰品的论文（FORMALISM_THEATER）。

类似地，ig 被分类为 intact/partial 但 er>=7 且 ei<=3 的论文，其信号也是矛盾的：分类器认为有完整推导链，但论文在认识论层面对理论解释完全不承诺，且高度依赖实验验证。这更可能是误分类或"关于实验方法的定理"，而非我们要的纯理论论文（EXPERIMENTAL_CLAIM）。

关键设计：**不是用单个负标志排除，而是用两个以上相互独立的负标志才排除**。单个信号的噪声很大；两个独立信号同时指向同一结论，可信度则大幅提升。

### 机制二：加权投票（维度相关结构）

PCA 分析揭示，mr/tn/md 两两相关 0.82-0.90，在主成分空间里几乎合并成同一轴（PC1 贡献 57.6% 的方差）。对这三个维度各自设阈值，本质上是对同一底层特质的三重惩罚。

正确的做法是把这三个维度视为对"数学骨架强度"这一单一信号的三次噪声测量，取平均后给一票。

相比之下，er 与 mr 强负相关（r≈-0.83）、ei 与主轴几乎正交（PCA PC3 10.7%），这两个维度是真正独立的信息轴，应当各自贡献一票。

投票框架的本质是：**用数据结构来决定维度的权重，而不是用直觉**。

### 机制三：组内相对排名

绝对阈值对所有论文用同样的切割线，而不管这篇论文在它所在的群体里处于什么位置。

一篇 partial 论文排在所有 partial 论文的第 92 百分位，比一篇 intact 论文排在所有 intact 论文的第 8 百分位更值得读。这个判断不依赖绝对分值，依赖的是局部分布信息——而局部分布信息在超矩形规则里是不可表达的。

实现上，对每个 ig 组独立计算 FLD 分的百分位排名，然后按组设置不同的百分位门槛（intact: top 60%，partial: top 38%）。这同时也捕捉了一个重要直觉：partial 组的基础质量低于 intact 组，因此相同百分位位置代表不同的绝对质量水平，需要不同的门槛。

## 第三阶段：MMR 的设计逻辑

规则筛选（无论 JSON 还是 Python）是在做**淘汰**：排除明显不要的。剩下的 2000-6000 篇论文里，还需要一个机制来决定精读哪 1500 篇。

排名（取 FLD 前 1500）会继续压缩多样性，因为 FLD 的方向被 intact/absent 质心差决定，本质上是 mr 的加权版本。结果：排名靠前的 1500 篇会是风格高度相同的统计学习理论论文。

MMR（Maximum Marginal Relevance）形式化了多样性：每一步选择"足够相关但与已选集合最不同"的论文。这在数学上是：

```
score(d) = λ · relevance(d) - (1-λ) · max_{d'∈S} similarity(d, d')
```

λ 偏低（~0.5）对应"覆盖采样"目标，而非"提纯最优"目标。

Monte Carlo 集成（100 次带噪声的独立 MMR 运行，Borda count 聚合）消解了贪心 MMR 的路径依赖问题——单次 MMR 的排名不稳定，但集合趋于稳定。

FLD 的循环逻辑问题（FLD 方向 ≈ ig 方向，而 ig 已经在规则层被过滤）通过引入 PCA 补偿解决：combined = 0.7×FLD + 0.3×PCA，其中 PCA 在规则过滤后的子集上计算，能捕捉到 FLD 低估的独立轴（ar, ei）。

## rescue_rules 的设计哲学

rescue 规则不是在补漏洞，而是在承认一个结构性事实：**任何单一打分轴都会系统性忽略某些类型的价值**。

每一条 rescue 规则背后是一类被主信号低估的论文：
- `THEORY_BREAKTHROUGH`（partial/broken 但 mr 和 tn 都极高）：推导未完成的理论开拓型论文，主信号因 ig 分类而低估它的贡献
- `DEEP_MATH_OUTLIER`（md≥9 无论 ig）：用到测度论、代数拓扑等深度工具的论文，即使论证不完整，工具本身就有精读价值
- `HIGH_EPISTEMIC`（ei≥9 且 mr≥5）：极端认识论承诺，主动寻求基础性解释的论文

这些是精读语料库的多样性保证，不是阈值微调的产物。

---

## 双盲验证的意义

三套独立规则集（rules_jes, rules_claude, rules_codex）对 rescue_rules 架构的独立收敛，是对这一设计哲学的结构性验证：

- 每套规则集都独立推导出"只靠主信号会系统性错过某些类型论文"的结论
- 每套规则集都独立地用 rescue 机制来覆盖这些论文类型
- 四套规则集的交集（961 篇）是高置信度核心语料库

这不是三个人都做了相同的事情——而是三个人从不同出发点独立地到达了相同的结构，说明这个结构是被数据和目标强迫出来的，而不是某个人的偏好。

## 设计决策的核心权衡

| 决策 | 我们选择 | 代价 | 获得 |
|---|---|---|---|
| 排除逻辑 + OR | 只要有规则命中就排除 | 相关维度会叠加惩罚同一底层特质 | 简单、可组合、易审查 |
| 保留逻辑 + rescue | 主动捞回被低估论文 | 需要手动定义每类边缘情况 | 多样性；不被单一信号绑架 |
| 不合并 mr/tn/md | 保留语义独立性 | 规则层面三维度冗余 | 边界案例（高tn低mr）有区分力 |
| MMR 而非排名 | 最大化覆盖多样性 | 排名不稳定（需Monte Carlo解决） | 语料库类型分布的广度 |
| FLD+PCA 而非纯 FLD | 补偿 FLD 对 ar/ei 的低估 | 计算复杂度增加 | 独立信息轴不被 FLD 吞没 |
