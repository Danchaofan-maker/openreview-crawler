# 打分模型对比实验报告

**日期**: 2026-05-10  
**数据规模**: C1-thinking 600篇 / C1-nothinking 49篇 / C3 50篇（重叠49篇）

---

## 实验设置

| 模型 | Prompt | Thinking | 样本数 | 数据文件 |
|------|--------|----------|--------|----------|
| C1-thinking (C1t) | prompt.md @ ea3589d (v0.7) | ON | 600 | `llm_v4pro_thinking_N600_seed42.jsonl` |
| C1-nothinking (C1nt) | prompt_c1_nothink.md (v0.7) | OFF | 49 | `llm_c1_nothink_N50_test.jsonl` |
| C3 | prompt_v0_8.md (v0.8) | OFF | 50 | `llm_c3_v08_N50.jsonl` |

**受控变量说明**：

- **C1t vs C1nt**：prompt 内容完全一致（diff 仅3处，均为 reasoning 通道的 operational 描述，不涉及任何评分标准）。唯一变量是 thinking mode。→ **干净的受控实验**。
- **C1nt vs C3**：thinking mode 一致（均为 OFF），但 prompt 从 v0.7 嵌套格式升级为 v0.8 扁平格式（字段缩写 + 四阶段输出协议）。存在两个变量（prompt 结构 + schema），不完全受控。

## 三、综合结论

### 变量隔离

```
C1t ──[thinking ON/OFF]──→ C1nt ──[prompt v0.7→v0.8]──→ C3
         ↑ 干净受控              ↑ 两变量混合，不完全受控
```

### 稳定维度（跨版本可信）

`mr`、`md`、`er` 在所有对比中偏差最小，可作为跨版本横向比较的基准维度。

### 不稳定维度

- **`ig`**：对 thinking mode 和 prompt 结构均高度敏感，不宜单独用于跨版本对比
- **`ar`**：在 absent 场景下两版本填值逻辑不同（C1t 填 null，C3 填具体值），std 极大
- **`tn`**：C1nt→C3 偏高 +1.45，疑与 v0.8 schema 改变输出路径有关

### C3 的 fuse 机制

C3 的 fuse=true（紧凑模式）率为 40%，低于 C1t 对应的 absent 率（约 40%，600篇中占比）。fuse 逻辑（`fuse = ig == "absent"`）机械正确率 100%，逻辑本身无误，差异来源于 ig 判定本身的版本漂移。

### 成本收益

| 场景 | 推荐方案 | 理由 |
|------|----------|------|
| 大规模筛选（>1000篇） | C3 | token 成本最低，mr/md/er 可信 |
| 需要 ig 精确判定 | C1t | thinking 模式 ig 判定更保守、更严格 |
| 快速基线对比 | C1nt | 成本低且与 C1t prompt 完全一致 |
